This guide walks you through cloning the repository, configuring your API keys, and submitting your first analysis. The whole process takes about five minutes.

<Tip>
  At minimum you need a **Gemini API key** to run TruthLens. Tavily and Azure AI Foundry are optional but strongly recommended — Tavily enables live web grounding, and Azure AI Foundry activates the Llama 3.1 fallback path.
</Tip>

## Prerequisites

- Node.js 18 or later
- npm
- A [Google AI Studio](https://aistudio.google.com) account (for Gemini)
- Optionally: a [Tavily](https://app.tavily.com) account and an [Azure AI Foundry](https://ai.azure.com) deployment

---

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the TruthLens repository and move into the project directory.

    ```bash
    git clone https://github.com/manavcodaty/TruthLens.git
    cd TruthLens
    ```
  </Step>

  <Step title="Install dependencies">
    Install all Node.js dependencies with npm.

    ```bash
    npm install
    ```

    This installs the core packages: `next`, `@google/generative-ai`, `@azure-rest/ai-inference`, `@tavily/core`, and the rest of the stack listed in `package.json`.
  </Step>

  <Step title="Create your environment file">
    Create a `.env.local` file in the root of the project and add your API keys.

    ```bash
    touch .env.local
    ```

    Then open it in your editor and add the following — replacing each placeholder with a real value:

    ```bash .env.local
    # Required — Gemini (primary analysis model)
    GEMINI_API_KEY=your_gemini_api_key_here
    GEMINI_MODEL=gemini-2.5-flash

    # Recommended — Tavily (live web grounding)
    TAVILY_API_KEY=your_tavily_api_key_here

    # Recommended — Azure AI Foundry (Llama 3.1 fallback)
    AZURE_AI_FOUNDRY_ENDPOINT=https://your-foundry-endpoint.inference.ai.azure.com
    AZURE_AI_FOUNDRY_API_KEY=your_foundry_api_key_here
    ```

    For a full explanation of every variable, including alternate names and which paths each key unlocks, see the [environment variables reference](/environment-setup).
  </Step>

  <Step title="Start the development server">
    Run the Next.js development server.

    ```bash
    npm run dev
    ```

    The server starts at `http://localhost:3000`. Open that address in your browser.
  </Step>

  <Step title="Open the analysis workspace">
    The TruthLens interface loads at `http://localhost:3000`. You will see an analysis workspace with a text input area.

    Paste any headline, article excerpt, or social post into the workspace.
  </Step>

  <Step title="Submit your first analysis">
    Submit the text and wait a few seconds while TruthLens:

    1. Builds a live web grounding pack using Tavily
    2. Runs credibility analysis with Gemini
    3. Falls back to Azure AI Foundry / Llama 3.1 if needed
    4. Returns a structured result

    The result panel shows the credibility score, a summary, extracted claims, warning flags, source notes, context notes, next steps, and any live citations retrieved during grounding.
  </Step>
</Steps>

## Testing the API directly

You can also submit text directly to the `POST /api/analyze` endpoint without using the UI.

### Example request

```bash
curl -X POST http://localhost:3000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"text": "BREAKING: Dubai will go into a full citywide curfew tonight after secret emergency meetings."}'
```

### Example response shape

```json
{
  "credibilityScore": "Mixed / use caution",
  "summary": "A concise overall explanation of the credibility assessment.",
  "mainClaims": [
    "Dubai will implement a full citywide curfew tonight.",
    "The decision followed secret emergency meetings."
  ],
  "flags": [
    {
      "type": "Emotional Language",
      "quote": "BREAKING",
      "explanation": "Uses urgency framing to pressure rapid sharing before verification."
    },
    {
      "type": "Weak Sourcing",
      "quote": "secret emergency meetings",
      "explanation": "Refers to meetings with no named attendees, organizations, or documents."
    }
  ],
  "sourceNotes": "No named officials or primary documents cited. Live grounding checked available sources.",
  "contextNotes": "Claim classified as time-sensitive: recency weighting was applied during retrieval.",
  "nextSteps": [
    "Check Dubai Media Office or UAE government official channels.",
    "Look for corroboration from established international outlets."
  ],
  "citations": [],
  "grounding": {
    "attempted": true,
    "succeeded": true,
    "searchQueries": [],
    "usedExtract": false,
    "warning": null,
    "retrievedAt": "2026-03-24T00:00:00.000Z",
    "isTimeSensitive": true
  },
  "warnings": []
}
```

The five possible `credibilityScore` values are:

| Score | Meaning |
|---|---|
| `Credible` | Well-supported by named sources and verifiable evidence |
| `Mostly credible` | Generally sound with minor gaps or caveats |
| `Mixed / use caution` | Contains a mix of supported and unsupported elements |
| `Weakly supported` | Claims lack adequate sourcing or corroboration |
| `High-risk / likely misleading` | Strong signals of manipulation, fabrication, or distortion |
